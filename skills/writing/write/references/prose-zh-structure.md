# 中文 prose 結構規則 (bold-period / list-dedensify / 段末收尾)

> 載入觸發: 中文 prose 中連續 3+ 個 `**xxx**。content` 起手段落, 或 bullets 多到像 PPT (≥5 個 bullet 連發或一段內 ≥3 個 list), 或段末出現「到這裡」「這說明」「也就是說」「可以看出」開頭的重述句.
> 來源: Waza skills/write/references/write-zh.md (bold + 句號節奏 / 列表去 list 化 / 段末收尾總結句)

中文技術 prose 的結構性 AI 味, 不是字詞層級, 是排版和段落收束方式. 三個高頻問題:

## 1. bold + 句號 vs 逗號

`**xxx**。content` 這種「小標題 + 句號 + 解釋」起手段落是 AI 模板感的高頻源.

**規則**: 不論密度, 預設換成 `**xxx**, content`, 讓 bold 變成句子的承重詞, 而不是獨立小標題. 整段讀起來是 prose, 不是工整 bold 列表.

```
NO: **alias**。我在 .zshrc 裡加了一行...
OK: **alias**, 我在 .zshrc 裡加了一行...
```

**例外**: `**xxx**` 本身就是一個完整句子 (如「**能跑不代表安全**」「**準不準才重要**」), 後面跟句號是正常句子結束, 不屬於「小標題」模式, 保留.

**判斷信號**: bold 部分單獨看是不是一個能獨立成立的句子.
- 是 (有主謂或完整斷言) → 句號
- 不是 (只是個標籤、短語、名詞) → 逗號

**對稱粗體起句殺傷力**: `**詞語**, ……` 連續出現 3 行以上即觸發 AI 模板感. 把第 4 個改成行內嵌入式粗體 (`句子裡加 **重點**`), 打破節奏.

## 2. 列表去 list 化漸進路徑

技術長文裡 bullet 列表過多會顯得像 PPT. 從 list 走到 prose 的漸進路徑:

1. **序號列表 1/2/3** → 改 bullets `- xxx` (去掉序號感)
2. **bullets `- **xxx**: content`** → 改 bold 段落 `**xxx**, content` (去掉 list marker)
3. **4+ 個 bold 段落** → 合成一個 prose 段落, 每項用分號串起來 (去掉 visual 切片感)

中間停在哪一步都行, 不必一步到底.

**判斷信號**:
- 每條都有大段獨立解釋 → 適合 bold 段落 (停在第 2 步)
- 每條 1 句話, 關係緊密 → 適合 inline prose 串起來 (走到第 3 步)
- 每條是平行規則, 讀者要掃一眼對照 → 保留 bullets 也行 (停在第 1 步)
- 章節標題已經把列表的總結寫出來了 → 內容部分盡量 prose 化

**序號 1/2/3 留給真正有順序的**: 優先級, 時間, 流程步驟, reference 排序. 不要拿來當「我有三點要說」的容器.

## 3. 段末收尾總結句 (高頻 AI tic)

一段剛解釋完某個機制, 結尾再加一句把剛才的意思重述一遍.

**識別信號**: 這類句子往往以以下開頭, 或是段落最後一句比其他句短且抽象:
- 「到這裡」
- 「這說明」
- 「這本身就是」
- 「也就是說」
- 「可以看出」
- 「歸根到底」
- 「換句話說」 (有時)

**範例**:
> NO: 「...具體解釋了 quota 機制. 這不只是設計選項, 是資源約束的結果.」← 末句重述
> NO: 「...模型在給下一代產出訓練數據. 模型本身成了訓練流水線裡的工具.」← 末句重述
> OK: 「...具體解釋了 quota 機制.」(說完就停)
> OK: 「...模型在給下一代產出訓練數據.」(事實本身已經說明一切)

**改法**: 說完就停, 事實本身已經說明一切, 不需要再「總結」一遍.

**例外不刪的情況**:
- **圖後說明段落**: 圖片的 alt text 讀者看不到, 圖後的文字說明才是讀者真正看到的解釋. 不屬於收尾總結.
- **章節間的模式切換信號**: 長文裡從原則轉向案例, 讀者需要知道內容性質變了. 例: 「前面幾節講的是原則, 這一節直接看 OpenClaw 怎麼落地.」這種有信息量, 留.
- **行動指引**: 結尾給讀者一個具體的下一步. 例: 「想知道自己的配置離這些原則差多遠, 跑一次 /health 是最快的方式.」

**判斷標準**: 句子裡有沒有對讀者有用的資訊. 有就留, 只在重述前文就刪.

## 自檢順序

掃 prose 時:

1. **bold 起手掃** (字符級): 連續 3+ 個 `**xxx**。` 起手 → flag, 4 個以上 → 強制至少改 1 個成 inline embedded
2. **bullet 掃** (字符級): bullet 數 ≥5 連發, 或一段內 ≥3 個 list block → flag, 提示 list-dedensify 路徑
3. **段末收尾掃** (要回讀): 每段最後一句, 用上面 7 個信號詞匹配, 命中後判斷有沒有信息量, 沒有就建議刪
