# 中文 Polish 後期殘留 6 類 checklist

> 載入觸發: draft 已經 polish 10+ 輪, 或使用者說「最後掃一遍」「都改差不多了」「再過一遍」, 或 edit 模式 + 該稿在會話中已經被 edit 過 ≥3 次.
> 來源: Waza skills/write/references/write-zh.md (Polish 後期殘留 checklist)

風格級 AI 味清乾淨後, 還有一類「句子級殘留」最容易被漏掉. 每輪 polish 末尾按這 6 類掃一遍.

## 1. 疊詞拗口 (字符級, 5 分鐘掃完)

同一句裡出現兩個意思接近的詞.

**識別**: 把句子拆成最小動作主謂賓, 看哪個修飾是冗餘的.

**常見**:
- 「也能上手用」← 「能用」「上手」其實是同一件事
- 「也都」← 兩個範圍詞
- 「也是另外」← 「另外」已經有「另一個」意思
- 「真正的最大問題」← 「真正」「最大」重複強調

**修法**: 留一個, 通常是更具體的那個.

## 2. 缺逗號長句 (字符級)

超過 25 字的句子裡沒有逗號斷點.

**識別**: 朗讀時想換氣的位置就是該加逗號的位置.

**常見漏點**:
- 時間狀語後 (「2026 年起所有」 → 「2026 年起, 所有」)
- 條件狀語後 (「如果 X 那麼 Y」 → 「如果 X, 那麼 Y」)
- 前置主題後 (「這個方案我們」 → 「這個方案, 我們」)

**修法**: 加逗號, 不要拆句. 拆句容易破壞節奏.

## 3. 中英術語一致性 (字符級)

同一篇裡同一個常用詞反覆切換中英形態.

**識別**: 搜文中所有英文 token, 看是否有對應中文已成主流, 主流就統一為中文.

**常見**:
- copy / 複製 (同篇兩種)
- context / 上下文
- setting / 設置
- preview / 預覽
- session / 會話 / 對話

**修法**: 全文搜該詞, 統一. 中文圈已有通用譯就用中文; 沒有 (如 prompt) 就保留英文.

## 4. 突然出現的比喻 / 概念 (要全文回讀)

作者自己腦子裡有的比喻 (「工作台」「水位」「帳本」「電梯」) 沒在前文鋪墊就直接用.

**識別**: 第一次出現的非通用比喻詞, 前一段必須有鋪墊, 沒有就換通用詞.

**常見漏點**:
- 第一次出現「工作台」, 但前文沒解釋為什麼把這東西叫工作台
- 第一次出現「水位」, 但讀者不知道在指什麼
- 第一次出現「帳本」, 但跟前文系統沒接上

**修法**: 要嘛在前一段加一句鋪墊, 要嘛換成通用詞 (「工作台」 → 「主要操作區」).

## 5.「上 / 下 / 裡」字殘留 (字符級)

「方案上都」「思路裡」「概念上」這類介詞殘留是早期草稿的痕跡, 刪掉不影響意思.

**識別**: 搜「上都」「裡都」「上的時候」「概念上」「思路上」.

**常見**:
- 「方案上都試過了」 → 「方案都試過了」
- 「概念上是這樣」 → 「概念是這樣」
- 「思路裡」 → 「思路」

**修法**: 直接刪掉介詞.

## 6. 缺關鍵修飾 (要全文回讀)

某個名詞第一次出現需要修飾才能區分性質, 漏了讀者會和上下文混淆.

**識別**: 每個產品 / 工具名第一次出現, 問一句「讀者能否區分這是誰家的、跟前文工具的關係」.

**常見**:
- 「Anthropic 官方推出的 Claude Design」 vs 「Claude Design」 (前者明確, 後者讀者不知道)
- 「Aave v3 lending protocol」 vs 「protocol」 (後者太空)
- 「我們前面說的 X 路線」 vs 「X 路線」 (後者不知道在說哪一個)

**修法**: 第一次出現加 1-3 字修飾, 後續可省.

## 執行順序建議

掃 prose 時, 順序是字符級 → 全文回讀:

1. **第一遍** (5 分鐘): 1, 2, 3, 5 — 字符級, 用 grep 模式掃
2. **第二遍** (要回讀): 4, 6 — 帶上下文判斷, 全文連讀

10 輪後 prose 大改不動了, 時間就花在這 6 類殘留上.

## 不要在早期 draft 用這個 checklist

這個 checklist 是 polish 後期的字符級工具. 早期 draft 該關注的是 voice / structure / 4-quadrant 命中, 不是疊詞和介詞殘留. 早期載入這個會讓 LLM 把注意力放錯地方.

判斷信號: 如果 prose 還有大段需要重寫的地方 (例如某段的 thesis 不清楚, 或某段在跟其他段重複), 不該動這個 checklist.
