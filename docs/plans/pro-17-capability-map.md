# PRO-17 — capability map for Claude Code / Codex / Hermes

> WHAT only. WHY lives in Linear PRO-17 plus brain decision `decisions/2026-06-15-pandastack-cli-orchestration-worker-protocol` and learning `learnings/architecture/2026-06-15-pandastack-architecture-cli-worker-bus-and-runtime`.
> This plan assumes PRO-15 is present on the branch: `scripts/pandastack doctor` already detects runtimes and prints next actions.

## Goal

Persist PRO-15 runtime detection into explicit capability-map files so pandastack skills and orchestration code can read a stable contract instead of probing Claude/Codex/Hermes ad hoc.

## Non-goals

- Do not migrate orchestration skills to consume the map everywhere. That is PRO-18. PRO-17 should provide the map and one small documented consumption helper/path.
- Do not create full new-user install docs. That is PRO-19.
- Do not add new AI runtime invocations outside `scripts/agent-worker`.
- Do not require Bun, Node, network, Linear, Claude, Codex, or Hermes for tests.
- Do not write secrets, tokens, or auth material into capability files. Auth state is boolean/status only.

## Desired state model

Three files define the map:

```text
pandastack.toml                         # repo/project desired config, committed
.pandastack/local/capabilities.json     # local detected state, gitignored
~/.pandastack/runtimes.json             # global runtime inventory, user-local
```

### `pandastack.toml`

Committed repo config. It should be small, human-readable, and stdlib-parseable by the existing CLI.

Minimum shape:

```toml
[capabilities]
schema_version = 1

[capabilities.roles.host]
runtimes = ["claude", "codex", "hermes"]

[capabilities.roles.worker]
backends = ["codex", "test"]

[capabilities.roles.operator]
runtimes = ["shell", "launchd", "hermes"]

[capabilities.paths]
local_state = ".pandastack/local/capabilities.json"
global_runtimes = "~/.pandastack/runtimes.json"
```

Keep this as desired/default config, not detected facts.

### `.pandastack/local/capabilities.json`

Gitignored local state generated by `scripts/pandastack doctor --write-capabilities` or equivalent. It should be deterministic enough for tests but may contain host-specific absolute paths where needed.

Minimum shape:

```json
{
  "schema_version": 1,
  "generated_at": "2026-06-18T00:00:00Z",
  "repo": "/abs/path/to/repo",
  "source": "scripts/pandastack doctor",
  "roles": {
    "host": { "claude": {}, "codex": {}, "hermes": {} },
    "worker": {},
    "operator": {}
  },
  "runtimes": {},
  "skills": {
    "counts": { "core": 24, "ext": 2, "total": 26 },
    "available": []
  },
  "auth": {
    "claude": { "status": "unknown|present|missing|unsupported" },
    "codex": { "status": "unknown|present|missing|unsupported" },
    "hermes": { "status": "unknown|present|missing|unsupported" }
  },
  "limits": {
    "network": "unknown|available|disabled",
    "write_scope": "read-only|workspace-write|unknown"
  },
  "next_actions": []
}
```

Do not include env var values, token paths, keychain entries, cookies, or raw CLI output that might leak secrets.

### `~/.pandastack/runtimes.json`

User-global runtime inventory generated by an explicit flag such as `scripts/pandastack doctor --write-runtimes` or `scripts/pandastack doctor --write-capabilities`. This is outside the repo and must not be required by tests except through a temp `HOME` fixture.

Minimum shape:

```json
{
  "schema_version": 1,
  "generated_at": "2026-06-18T00:00:00Z",
  "runtimes": {
    "claude": { "present": true, "path": "/usr/local/bin/claude", "auth": "unknown" },
    "codex": { "present": true, "path": "/usr/local/bin/codex", "auth": "unknown" },
    "hermes": { "present": false, "path": null, "auth": "unsupported" }
  }
}
```

## Tasks

### pro-17-T01 — Add committed desired config

- scope: `pandastack.toml` new root file.
- behavior:
  - declare schema version, role names, desired host runtimes, worker backends, operator runtimes, and state file paths.
  - keep comments concise and public-safe.
- acceptance:
  - `test -f pandastack.toml`
  - `python3 - <<'PY'` can parse the config using `tomllib` on Python 3.11+ or the repo's minimal parser fallback.
- depends-on: none
- status: planned

### pro-17-T02 — Gitignore local state

- scope: `.gitignore`.
- behavior:
  - ignore `.pandastack/local/` or at least `.pandastack/local/capabilities.json`.
  - do not ignore committed repo config `pandastack.toml`.
- acceptance:
  - `git check-ignore .pandastack/local/capabilities.json` exits 0.
  - `git check-ignore pandastack.toml` exits nonzero.
- depends-on: pro-17-T01
- status: planned

### pro-17-T03 — Extend `scripts/pandastack doctor` to emit/write capability map

- scope: `scripts/pandastack`, optional stdlib helper extraction if needed.
- behavior:
  - keep existing PRO-15 `doctor` and `init` behavior compatible.
  - add `scripts/pandastack doctor --capabilities-json` to print the capability-map JSON schema.
  - add `scripts/pandastack doctor --write-capabilities` to write `.pandastack/local/capabilities.json` and `~/.pandastack/runtimes.json` unless a narrower flag shape is simpler and tested.
  - expose generated paths in normal `doctor --json` output, but do not require local files to exist before writing.
  - support deterministic tests with an env var such as `PANDASTACK_NOW` for generated timestamps.
- acceptance:
  - `scripts/pandastack doctor --capabilities-json | python3 -m json.tool >/dev/null`
  - `PANDASTACK_NOW=2026-06-18T00:00:00Z HOME=<tmp> scripts/pandastack doctor --write-capabilities --json` writes both files under repo temp/local and temp HOME.
  - generated JSON includes `roles.host`, `roles.worker`, `roles.operator`, `skills.counts`, `auth`, `limits`, and `next_actions`.
- depends-on: pro-17-T01, pro-17-T02
- status: planned

### pro-17-T04 — Add a read helper for skills/orchestration

- If a local generated file is created during manual testing, clean it up before committing unless it is intentionally ignored and not staged.
