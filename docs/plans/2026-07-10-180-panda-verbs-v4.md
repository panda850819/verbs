Product definition:

> An opinionated skill pack for taking software work from ambiguity to verified delivery.

Hero:

> Hard-won ways of working, encoded as composable skills for coding agents.

Support statement:

> Verified on Claude Code and Codex. Hermes supports selective manual import.

Panda Verbs owns skills, shared procedural primitives, dispatch, narrow host adapters, installation, manifests, evals, tests, and release validation. Identity, context, brain/state, project SSOT, runtimes/models, scheduling, autonomous drivers, connectors, and global model routing stay outside.

Living surfaces may use `pandastack` only in an explicit v3-to-v4 migration sentence or compatibility shim. `CHANGELOG.md` history, `archive/`, dated `docs/plans/`, dated audits/evals, `ROADMAP.md` history, and source-attribution notes keep historical names.

Forbidden living claims: `operator OS`, `personal-os`, `one substrate`, `three runtimes`, `no vendor lock-in`, `interchangeable runtimes`, retired lifecycle lists, and any claim that Hermes has packaged parity.

## Compatibility contract

- `/verbs:*` is the only canonical v4 plugin namespace. Host plugin systems do not guarantee a second namespace, so no fake `/pandastack:*` alias is promised.
- `scripts/pandastack` remains the only RC shim. It forwards to `scripts/verbs`, maps the two legacy repo/manifest environment overrides, and prints one deprecation line to stderr. The retired state CLI has no v4 replacement.
- New `PANDA_VERBS_*` variables win. The opt-in verify adapter keeps the RC fallback `PANDASTACK_VERIFY_GATE`; test-only/internal knobs switch directly to the new prefix. State/private-overlay fallbacks were removed with those out-of-scope features.
- Existing `~/.pandastack` lifecycle data is left untouched. v4 does not read, move, or delete it.
- GitHub redirects old repository URLs after rename. Install docs still give explicit uninstall/reinstall commands because plugin namespace and cache roots change.
- Do not recreate `panda850819/pandastack`; reusing the old repository name would break GitHub's redirect.
